SIMILARITIES BETWEEN LET ,VAR AND CONST


  1. Block Scope: All three can be used within a block (e.g., within {}), but their scoping rules differ. let and const are block-scoped, while var is function-scoped.
  2. Hoisting: All three declarations are hoisted to the top of their containing scope. However, while var can be used before it's declared (with an undefined value), let and const cannot be accessed before their declaration, resulting in a "temporal dead zone."
  3. Re-declaration: var allows re-declaration within the same scope, while let and const do not allow re-declaration in the same block scope.
  4. Variable Initialization: All three can be initialized at the time of declaration or later in the code. However, const requires an initializer at the time of declaration.
  5. Use in Global Scope: When declared in the global scope, all three can create properties on the global object (e.g., window in browsers). However, var creates a property, while let and const do not.

DIFFERENCES BETWEEN LET ,VAR AND CONST


  1. Scope:
    • var: Function-scoped or globally scoped, meaning it is accessible throughout the entire function or globally if declared outside any function.
    • let: Block-scoped, meaning it is only accessible within the nearest enclosing block (e.g., inside {}).
    • const: Also block-scoped, similar to let.
  2. Re-declaration:
    • var: Allows re-declaration of the same variable within the same scope.
    • let: Does not allow re-declaration in the same block scope.
    • const: Does not allow re-declaration in the same block scope.
  3. Initialization:
    • var: Can be declared without initialization and can be assigned a value later.
    • let: Can also be declared without initialization and assigned a value later.
    • const: Must be initialized at the time of declaration; otherwise, it results in a syntax error.
  4. Re-assignment:
    • var: Can be re-assigned a new value after its initial assignment.
    • let: Can also be re-assigned a new value.
    • const: Cannot be re-assigned; once a value is assigned, it cannot be changed (though properties of objects assigned to a const can be mutated).

SHORT HISTORY OF JAVASCRIPT (ECMA SCRIPT 1) TO (ES6)


JavaScript was created in 1995 by Brendan Eich while working at Netscape. The first edition of ECMAScript (ES1) was standardized in June 1997 to provide a consistent scripting language for web development. ES2 followed in 1998, focusing on minor updates and maintenance.

In 1999, ES3 introduced significant features like regular expressions, try/catch, and more robust string manipulation. After a long hiatus, ES4 was proposed but ultimately abandoned due to complexity and disagreement among stakeholders.

In 2009, ES5 was released, adding "strict mode," JSON support, and new array methods. This revitalized interest in JavaScript.
